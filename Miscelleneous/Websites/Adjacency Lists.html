Graphs have many applications including the analysis of electrical circuits, creation of databases, finding the shortest distance between two points, and project planning.

Graph Definitions

A graph is defined by:
  1. A set of vertices or nodes.
  2. A set of edges or arcs.
Generally, the vertices and edges are used to describe undirected graphs, while nodes and arcs are used to describe directed graphs.

An undirected graph uses a edge to connect two vertices, e.g, (v0, v1). A path may go from v0 to v1 or from v1 to v 0.

A directed graph, or digraph, uses an arc to connect two nodes. This arc specifies direction. For example, the arc, <v2, v 1>, indicates that v2 -> v1. Thus, v 2 is the tail of the arc and v1 is the head of the arc.

Graph Restrictions

Graph definitions carry the following restrictions:

  1. For any edge, (vi, vj) or arc <vi, vj>, i, does not equal j.
  2. Two vertices or nodes, vi and vj, may be connected by only one edge (vi - vj) in an undirected graph or two arcs of type: vi->vj and vj->vi.


Graph Terminology

  1. The degree of a vertex or node is the number of edges or arcs that are incident on the vertex or arc.

    For digraphs:

    1. The out degree of a node is the number of arcs that have the node as their tail.
    2. The in degree of a node is the number of arcs that have the node as their head.


  2. A complete undirected graph has: 
    n(n-1)

    edges, where n represents the number of vertices.

There are three common graph representations:
  1. Matrix Representation. A standard two dimensional array may be used to represent a graph. The rows represent the first vertex in an ordered pair and the columns represent the second vertex in the ordered pair. A zero in a cell indicates that there is no edge between the two vertices while a one indicates that there is an edge. For undirected graphs, an edge (vi, vj) there will be a 1 in cells: Matrix[i][j] and Matrix[j][i]. For directed graphs, there will be a 1 only in cell Matrix[i][j].

    The get_node() Function

    When we create the adjacency lists for an undirected graph, we must add each of vertex to the other's adjacency list. For example, if we have an edge, (v0, v1, we must add vertex 1 to vertex 0's adjacency list and we must add vertex 0 to vertex 1's adjacency list. Each addition requires the allocation of dynamic memory. Therefore, we use a get_node() function to assist in memory allocation. The code for this function appears in the following table.

    The get_node() function
    node_pointer get_node()
{
 /* get a new node, check to see if memory is full */
 node_pointer ptr = (node_pointer) malloc(sizeof(struct node));
 if (!ptr) {
   fprintf(stderr, "The memory is full\n");
   exit(1);
 }
 return ptr;
}


    The create_list() Function

    The create_list() function allows the user to enter pairs of vertices as long as the vertices are greater than or equal to zero. Each of the vertices is added to the others adjacency list. It is assume that the user will choose vertices in order, that is, if the user requires three vertices she or he will number them 0,1,2. The function counts the vertices when the user has finished. The code for the create_list() function appears in the following table.

    The create_list function 
    void create_list(node_pointer graph[], int *size)
/* create the adjacency list */
{
   int i, vertex1, vertex2;
   node_pointer temp;

  for (i = 0; i < MAX_NODES; i++)
     graph[i] = NULL;

  printf("Enter a pair of vertices, -1 -1 to quit: ");
  scanf("%d%d",&vertex1,&vertex2);
  while (vertex1 >= 0)  {
    /* place vertex 1 in vertex 2's adjacency list */
     temp = get_node();
     temp->vertex = vertex1;
     temp->link = graph[vertex2];
     graph[vertex2] = temp;

     /* place vertex 2  in vertex 1's adjacency list */
     temp = get_node();
     temp-> vertex = vertex2;
     temp->link = graph[vertex1];
     graph[vertex1]  = temp;
     printf("Enter a pair of vertices, -1 -1 to quit: ");
     scanf("%d%d",&vertex1,&vertex2);
   }
   i = 0;
   for (temp = graph[i]; temp; temp = graph[++i])
       ;
   *size = i;
}

    The print_list() function

    The print_list() function prints our the adjacency list for each of the vertices. This function uses a for statement to cycle through each of the vertices in use. A while statement within the for statement is used to traverse the adjacency list for a vertex.

    The code for the print_list() function appears in the following table.

    The print_list() function 
    void print_list( node_pointer graph[], int size)
/* print out the adjacency list by the first vertex */
{
   int i;
   node_pointer temp;

   printf("\n%10s%20s","vertex","Adjacency List\n\n");

   for (i = 0; i < size; i++){
      printf("%5d%15s",i," ");
      temp = graph[i];
      while (temp) {
	 printf("%3d",temp->vertex);
	 temp =  temp->link;
       }
       printf("\n");
    }
    printf("\n\n");
}
